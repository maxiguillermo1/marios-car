# marios-car

I want to build a web app to organize and manage maintenance for all of my cars.

This should be a real-life personal maintenance app, not just a demo. Keep it as a web app for now. I have photos of my cars that I want the app to use, and I also have spreadsheets for each car that contain maintenance history.

Each car profile should include:
- Car photo
- Make / model
- Mileage
- Maintenance history
- Maintenance schedule
- Job type
- Date of service
- Notes
- Vendor / shop used

The goal is to replace my messy spreadsheets with a cleaner, more organized app where I can easily see each car, track past maintenance, plan future maintenance, and keep notes/vendors connected to each job.

Start by building a clean dashboard where I can view all my cars as cards with photos. When I click a car, it should open a detailed page for that vehicle with its maintenance records and schedule.

Use placeholder data for now, but structure the app so I can later import my spreadsheet data and upload real car photos. We can go deeper into fields, reminders, vendors, and spreadsheet importing later.

Prioritize:
- Clean modern UI
- Easy navigation
- Mobile-friendly design
- Simple data structure
- Scalable setup for adding more cars later
- Clear separation between cars, maintenance records, vendors, and schedules
